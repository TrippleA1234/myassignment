Differentiate between NULL and UNDIFINED data type in javascript

UNDEFINED

  1. Definition: A variable is of type undefined when it has been declared but not assigned a value. It is also the default value for uninitialized variables.
  2. Usage: It typically represents the absence of a value or the lack of assignment.
  3. Properties:
    1. undefined is a global object property.
    2. typeof undefined returns "undefined".
  4. Common use cases:
    1. Function arguments not passed a value will be undefined.
    2. Accessing an object property that doesnâ€™t exist returns undefined.
    5. NULL
    1. Definition: null is a special object value that represents the intentional absence of any object value. It is explicitly assigned to a variable to indicate that it should be empty or "nullified".
    2. Usage: It is used when you want to assign an empty or "no value" state to a variable, typically to represent the absence of an object.
  5. Properties:
    1. null is an object (this is considered a JavaScript quirk).
    2. typeof null returns "object", which is a known bug in JavaScript.
